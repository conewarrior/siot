B2C와 B2B 디자인은 다릅니다. 셀렉트스타에서 데이터 가공 플랫폼과 어드민 툴을 디자인하면서 배운 B2B SaaS만의 특수성에 대해 이야기합니다.

## B2B는 왜 다른가

B2C 서비스는 첫인상이 중요합니다. 사용자가 5초 안에 떠날 수 있기 때문입니다. 하지만 B2B는 다릅니다. 사용자는 이미 "써야 하는" 상황에 있습니다. 회사가 선택한 툴이니까요.

이 차이가 디자인 접근법을 완전히 바꿉니다:

| B2C | B2B |
|-----|-----|
| 첫인상 중요 | 학습 곡선 허용 |
| 단순함 추구 | 기능 밀도 높음 |
| 감성적 연결 | 효율성 우선 |
| 짧은 세션 | 긴 세션 |

## 복잡한 정보구조(IA) 설계

셀렉트스타의 데이터 가공 플랫폼은 복잡한 워크플로우를 가지고 있습니다:

1. 프로젝트 생성
2. 데이터 업로드
3. 가공 규칙 설정
4. 작업자 배정
5. 품질 검수
6. 결과물 다운로드

각 단계마다 수십 개의 옵션과 설정이 있습니다. 이걸 어떻게 정리할 것인가?

### 원칙 1: 점진적 공개 (Progressive Disclosure)

모든 옵션을 한 번에 보여주면 사용자는 압도당합니다. 필수적인 것만 먼저 보여주고, 고급 옵션은 숨겨둡니다.

```
[기본 설정]
- 프로젝트 이름: ________
- 데이터 유형: 이미지 ▼

[고급 설정 ▶]  ← 클릭 시 펼쳐짐
```

### 원칙 2: 컨텍스트 유지

B2B 사용자는 여러 프로젝트를 동시에 관리합니다. 현재 어디에 있는지 항상 알 수 있어야 합니다.

- 브레드크럼 네비게이션 필수
- 사이드바에 현재 프로젝트 상태 표시
- 최근 작업 히스토리 제공

### 원칙 3: 벌크 액션 지원

B2B에서는 100개의 아이템을 하나씩 처리하는 것이 아니라, 한 번에 처리해야 합니다.

- 전체 선택 / 개별 선택
- 필터링 후 일괄 적용
- CSV 업로드/다운로드

## 어드민 툴 디자인

내부 직원이 사용하는 어드민 툴은 또 다른 챌린지입니다. "어차피 우리끼리 쓰는 거니까" 대충 만들면 안 됩니다.

### 인하우스도 사용자다

셀렉트스타에서 계약서 관리 페이지를 개선할 때, 실제 담당자를 인터뷰했습니다. 그들의 워크플로우를 관찰하니 비효율이 눈에 보였습니다:

- 엑셀에서 복사 → 어드민에 붙여넣기 → 다시 엑셀로 정리
- 같은 정보를 3군데에 입력
- 상태 변경할 때마다 페이지 새로고침

이런 비효율을 하나씩 제거하니 담당자의 업무 시간이 크게 줄었습니다.

## 클라이언트향 SaaS 전환

이미지 라벨링 툴을 SaaS로 전환하는 프로젝트를 리드했습니다. 기존에는 우리 작업자만 사용하던 툴을 클라이언트가 직접 사용할 수 있도록 개방하는 것이었습니다.

### 멀티테넌시 고려

여러 회사가 같은 시스템을 사용하므로:

- 권한 관리가 복잡해짐
- 데이터 격리가 필수
- 화이트라벨링 요구

### 셀프서비스 온보딩

B2B SaaS의 핵심은 셀프서비스입니다. 영업 담당자 없이도 고객이 스스로 가입하고, 설정하고, 사용할 수 있어야 합니다.

- 단계별 온보딩 위자드
- 인터랙티브 튜토리얼
- 컨텍스트 도움말

## 배운 것들

### 1. 효율성이 미학이다

B2B에서 "예쁜" 디자인은 "빠른" 디자인입니다. 클릭 수를 줄이고, 로딩을 없애고, 단축키를 제공하세요.

### 2. 파워 유저를 위해 디자인하라

B2B 사용자는 매일 8시간씩 당신의 제품을 사용합니다. 초보자를 위한 단순함보다, 숙련자를 위한 효율성이 중요합니다.

### 3. 에러 방지가 에러 메시지보다 낫다

복잡한 워크플로우에서 실수는 비용입니다. 실수할 수 없는 구조를 만드세요. 위험한 액션은 확인 단계를 추가하고, 되돌리기 기능을 제공하세요.

B2B SaaS 디자인은 화려하지 않습니다. 하지만 잘 만들어진 B2B 도구는 사용자의 하루를 바꿉니다. 그리고 그것이 진짜 좋은 디자인입니다.
