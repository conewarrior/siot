## 여러 컴퓨터에서 같은 커맨드를 쓰고 싶었다

Claude Code를 쓰다 보면 자주 쓰는 작업을 커맨드로 만들게 된다. `/cleanup-ui`처럼 UI 코드 정리하는 커맨드, `/commit` 같은 git 워크플로우 커맨드 등이 그렇다.

이런 커맨드들을 여러 컴퓨터에서 똑같이 쓰고 싶을 때가 있다. 집 컴퓨터에서 만든 커맨드를 회사에서도 쓰고 싶은데, 매번 파일을 복사하기는 번거로웠다.

처음에는 `claude-toolkit`이라는 저장소를 만들어서 커맨드들을 모아뒀다.

```
claude-toolkit/
├── commands/
│   ├── cleanup-ui.md
│   ├── install-command.md
│   ├── install-skill.md
│   ├── install-agent.md
│   └── ...
├── skills/
│   ├── pdf/
│   ├── pptx/
│   ├── frontend-design/
│   └── ...
├── hooks/
├── agents/
└── README.md
```

새 컴퓨터에서 커맨드를 설치하려면 저장소를 클론하고, 원하는 커맨드 파일을 찾아서 `.claude/commands/`에 복사해야 했다. 새 커맨드를 만들어서 공유하려면 로컬 저장소로 이동해서 파일을 복사한 뒤 git add, commit, push를 해야 했다.

```bash
# 기존: 저장소 클론 후 수동 복사
cd ~/dev/claude-toolkit
git pull origin main
cp commands/cleanup-ui.md ~/my-project/.claude/commands/
```

동작은 하지만 번거로웠다. 특히 저장소를 클론해둔 경로를 기억해야 하고, 항상 그 폴더가 최신 상태인지도 신경 써야 했다.

## GitHub API로 해결하기

어느 날 Claude에게 "GitHub API로 파일을 직접 가져올 수 있는지" 물어봤다. `gh` CLI가 GitHub API를 래핑해서 제공하고 있었다.

```bash
# 파일 내용 가져오기
gh api repos/owner/repo/contents/path/to/file.md \
  --jq '.content' | base64 -d

# 파일 업로드하기
gh api repos/owner/repo/contents/path/to/file.md \
  -X PUT \
  -f message="Add file" \
  -f content="$(base64 < local-file.md)"
```

이 방식이라면 로컬에 저장소를 클론해두지 않아도 된다. 인터넷만 되면 어디서든 똑같이 동작한다.

GitHub API를 활용해서 커맨드 관리 시스템을 만들었다.

| 용도 | 커맨드 | 설명 |
|------|--------|------|
| 설치 | `/install-command` | 프로젝트에 커맨드 설치 |
| 설치 | `/install-skill` | 프로젝트에 스킬 설치 + hook 등록 |
| 설치 | `/global-command` | 전역(`~/.claude/`)에 커맨드 설치 |
| 배포 | `/publish-command` | 커맨드를 GitHub에 업로드 |
| 배포 | `/publish-skill` | 스킬 폴더를 GitHub에 업로드 |

이제 새 컴퓨터에서 커맨드를 설치하려면 `/install-command cleanup-ui`만 실행하면 된다. 저장소 클론도, 경로 찾기도, 파일 복사도 필요 없다.

## 마켓플레이스와 함께 쓰기

Claude Code에는 공식 플러그인 마켓플레이스가 있다. 둘은 용도가 다르다.

| | 마켓플레이스 | 개인 GitHub 저장소 |
|---|---|---|
| **대상** | 공개 배포 (누구나 설치) | 개인/팀 전용 |
| **비공개 커맨드** | 불가 | 가능 |
| **업데이트** | 버전 관리 필요 | push하면 바로 반영 |
| **설정** | `marketplace.json` 작성 | 파일만 있으면 됨 |

`/commit`, `/review-pr` 같은 범용 커맨드는 마켓플레이스가 적합하고, `/cleanup-ui`, `/deploy-staging` 같은 개인/팀 워크플로우는 GitHub 저장소가 더 맞다.

마켓플레이스는 새 컴퓨터에서 전체 셋업할 때 유용하고, GitHub API는 개별 커맨드를 빠르게 설치/업데이트할 때 유용하다. 그래서 claude-toolkit 저장소에 마켓플레이스 구조를 추가했다.

```
claude-toolkit/
├── .claude-plugin/
│   ├── marketplace.json    # 마켓플레이스 정의
│   └── plugin.json         # 플러그인 메타데이터
├── commands/
└── skills/
```

`/publish-command`를 수정해서 파일 업로드할 때 `marketplace.json`의 버전도 자동으로 올리도록 했다. 이제 다른 컴퓨터에서 `/plugin marketplace update`를 실행하면 새 버전이 반영된다.

## 플러그인 네임스페이스 문제와 해결

마켓플레이스로 플러그인을 설치하고 나서 불편한 점을 발견했다. 플러그인으로 설치하면 `플러그인명:커맨드명` 형식으로만 호출할 수 있다. `/install-command`만 치고 싶은데 `/toolkit-commands:install-command`를 쳐야 한다.

여러 플러그인이 같은 이름의 커맨드를 가질 수 있으니까 네임스페이스로 구분하는 건 합리적이다. 하지만 자주 쓰는 커맨드에 매번 긴 접두사를 붙이기는 번거롭다.

Claude Code에서 커맨드를 인식하는 경로는 두 가지다.

| 경로 | 호출 방식 |
|------|-----------|
| 플러그인 (`~/.claude/plugins/...`) | `플러그인명:커맨드명` |
| 전역 commands (`~/.claude/commands/`) | `커맨드명` |

플러그인의 커맨드 파일들을 전역 commands 폴더에 복사하면 접두사 없이 바로 쓸 수 있다. 매번 수동으로 복사하기 번거로워서 `/global-plugin` 커맨드를 만들었다.

```bash
/global-plugin                      # 설치된 플러그인 목록
/global-plugin toolkit-commands     # 해당 플러그인을 전역으로 설치
```

결국 세 가지 설치 방식이 생겼다.

| 방식 | 용도 | 접두사 |
|------|------|--------|
| `/install-command` | 개별 커맨드 설치 | 없음 |
| `/plugin install` | 플러그인 전체 설치 | 필요 |
| `/global-plugin` | 플러그인 → 전역 승격 | 없음 |

개인적으로는 플러그인으로 설치한 뒤 바로 `/global-plugin`으로 전역에 복사하는 방식이 가장 편하다. 마켓플레이스의 버전 관리 기능도 쓰면서 호출할 때는 짧은 이름을 쓸 수 있다.

추가로 `/publish-command`와 `/publish-skill`도 수정해서 로컬에 없으면 전역을 확인하도록 했다. 전역 폴더에서 커맨드를 수정해도 바로 퍼블리시할 수 있다.

```
탐색 순서:
1. .claude/commands/$name.md (프로젝트 로컬)
2. ~/.claude/commands/$name.md (전역)
```
