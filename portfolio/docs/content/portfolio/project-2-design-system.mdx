# 디자인 시스템

## 프로젝트 개요

확장 가능한 컬러 시스템과 컴포넌트 라이브러리 구축

### 역할
- 디자인 시스템 아키텍처
- 토큰 설계
- 문서화

### 기간
2024년 - 현재

---

## 문제 정의

기존 디자인 시스템의 한계:

- 다크모드 대응의 어려움
- 색상 일관성 부재
- Figma-코드 간 동기화 문제

---

## 해결 과정

1. **컬러 시스템 재설계**
   - Lightness 기반 색상 스케일 구축
   - 의미론적 토큰 체계 정의

2. **Figma Variables 연동**
   - 디자인 토큰 자동 동기화
   - 모드(Light/Dark) 전환 시스템

3. **컴포넌트 라이브러리**
   - 재사용 가능한 기본 컴포넌트
   - 접근성(a11y) 가이드라인 적용

---

## 결과

- 디자인-개발 핸드오프 시간 **50% 단축**
- 컴포넌트 재사용률 **80%** 달성
- 다크모드 전환 완벽 지원

---

## 회고

### 잘한 점
- 체계적인 토큰 구조로 확장성 확보
- 문서화를 통한 팀 온보딩 개선

### 개선할 점
- 더 다양한 컴포넌트 패턴 필요
- 테스트 자동화 강화
