# 인트라넷 UI Flow 개선

## PM들을 위한 UI Flow 개선

PM들의 미션 관리 워크플로우를 단순화한 인트라넷 UX 개선 프로젝트

### 회사
셀렉트스타

### 기간
2022.12 - 2023.06

### 역할
- 프로덕트 디자이너 (전체 과정)
- User Flow 분석
- 정보구조 재설계
- UI 설계

---

## 기존 시스템의 문제

PM들이 미션을 설정하고 관리하는 과정에서 발생한 문제점들

### 기능 분산
- 미션 설정, 대시보드 기능이 여러 페이지에 분산
- 한 작업을 위해 3-4개 페이지를 오가야 함

### 카테고리 관리의 복잡성
- 카테고리 설정이 하위 위계에 숨어 있음
- 설정 변경 시 찾기 어려움

### 데이터 현황 파악 어려움
- 수집 데이터와 검수 데이터 현황을 따로 확인
- 전체 진행 상황을 한눈에 볼 수 없음

---

## AS-IS 분석

기존 User Flow 분석을 통해 5가지 핵심 문제점 도출


### 도출된 문제점

1. **깊은 뎁스 구조**: 핵심 설정이 3-4단계 하위에 위치
2. **분산된 기능**: 연관 기능이 서로 다른 메뉴에 분리
3. **중복 진입점**: 동일 기능에 여러 경로로 접근
4. **불명확한 위계**: 카테고리와 미션의 관계가 모호
5. **정보 과부하**: 한 화면에 불필요한 정보 다수 노출

---

## TO-BE 설계

분석 결과를 바탕으로 정보구조 재설계


### 개선 방향

#### 카테고리 설정을 최상단 위계로
- 카테고리 → 미션 → 세부설정 순서로 위계 명확화
- 설정 진입점 단일화

#### 기능 통폐합
- 분산된 미션 관리 기능을 하나의 허브로 통합
- 연관 기능 그룹핑

#### Overview로 한눈에 확인
- 수집/검수 데이터 현황을 대시보드 형태로 통합
- 주요 지표 실시간 표시

---

## 결과

### User Flow 단축
- AS-IS → TO-BE: **35단계로 축소**

### 정보구조 단순화
- 메뉴 뎁스 4단계 → 2단계
- 진입점 7개 → 3개로 통합

### PM 작업 효율 향상
- 미션 설정 시간 단축
- 데이터 현황 파악 즉시 가능

## 회고

### 잘한 점
- 위계 재설계로 정보구조 단순화
- User Flow 분석을 통한 문제 정량화
- PM 인터뷰 기반의 실제 페인포인트 발굴

### 개선할 점
- 개선 후 정량적 효과 측정 부재
- A/B 테스트 없이 전면 적용
- 기존 사용자의 학습 곡선 고려 부족
