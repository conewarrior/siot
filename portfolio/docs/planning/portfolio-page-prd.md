## 개요

현재 포트폴리오 사이트에 **UX 디자인 포트폴리오** 전용 페이지를 추가한다.

- **메뉴 위치**: 프로젝트 | **포트폴리오** | 소개
- **URL**: `/portfolio`
- **SEO**: `noindex` 처리 (검색엔진 노출 방지, 링크로만 접근)

## 페이지 구조 (레이아웃) - v2

### 데스크톱 (스크롤 스냅)

```
┌─────────────────────────────────────────────────────────────┐
│  [로고]                          [프로젝트] [포트폴리오] [소개]  │  ← 기존 헤더 유지
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌──────────┐  ┌─────────────────────────────────────────┐ │
│  │ 00       │  │                                         │ │
│  │ [흰색]   │  │                                         │ │
│  │ Cover    │  │        100vh 슬라이드 영역               │ │
│  ├──────────┤  │        (스크롤 스냅)                     │ │
│  │ 01       │  │                                         │ │
│  │ [오렌지] │● │        - 각 슬라이드가 화면 높이 차지     │ │
│  │ 프로젝트1 │  │        - 스크롤 시 다음 슬라이드에 스냅   │ │
│  │          │  │        - 섹션별 배경색 변경              │ │
│  │ (확장됨) │  │                                         │ │
│  │          │  │                                         │ │
│  ├──────────┤  ├─────────────────────────────────────────┤ │
│  │ 02       │  │  ━━━━━━━━━●━━━━━━━   [PDF 다운로드]    │ │
│  │ [보라]   │  │  (프로그레스 바)                        │ │
│  └──────────┘  └─────────────────────────────────────────┘ │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**레이아웃 특징**:
- **헤더**: 기존 사이트 헤더 그대로 표시 (다른 페이지로 이동 가능)
- **좌측 네비게이션**: 화면 고정(fixed), 스크롤 진행률에 따라 현재 섹션 카드 확장
- **메인 영역**: 100vh 단위 슬라이드, 스크롤 스냅(scroll-snap-type: y mandatory)
- **하단 인디케이터**: 프로그레스 바 형태 + PDF 다운로드 버튼

### 좌측 네비게이션 동작

1. **카드 확장**: 현재 보이는 섹션의 카드가 세로로 확장됨 (스크롤 진행률 반영)
2. **활성 인디케이터**: 확장된 카드 우측에 작은 점(●) 표시
3. **클릭 동작**: 해당 섹션 시작점으로 부드럽게 스크롤

### 모바일

- 좌측 네비게이션 숨김
- 순수하게 세로 스크롤만 지원
- 헤더는 그대로 표시

## 슬라이드 구성

### 전체 순서

```
1. 커버 (+ 목차 통합)
2. 프로젝트 1 (4-10장)
3. 프로젝트 2 (4-10장)
4. 프로젝트 3 (4-10장)
5. About Me (포트폴리오 전용 소개)
6. 에필로그 (감사 인사 + 연락처)
```

**프로젝트 순서**: 가장 자신있는 것 먼저

### 케이스 스터디 필수 섹션 (각 프로젝트당)

| 섹션 | 설명 | 예상 슬라이드 |
|------|------|--------------|
| **Problem + Context** | 무슨 문제였고, 왜 중요했는지 | 1장 |
| **Process + Decision** | 어떤 선택지가 있었고 왜 그걸 골랐는지 | 2-4장 |
| **Outcome + Metrics** | 결과물과 수치화된 성과 (표로 시각화) | 1-2장 |
| **Reflection + Learning** | 배운 점, 다르게 했을 것 | 1장 |

### 콘텐츠 형태

- **시각 자료 중심** + 충분한 설명
- **지표는 표로 시각화**
- 블로그 글과 PDF 프로젝트 모두 동일한 케이스 스터디 구조로 재구성

## 네비게이션 - v2

### 페이지 전환 방식 (데스크톱)

1. **세로 스크롤** - 마우스 휠 또는 트랙패드로 스크롤
2. **스크롤 스냅** - 각 슬라이드(100vh)에 자동 스냅
3. **좌측 카드 클릭** - 해당 섹션으로 부드럽게 스크롤

### 좌측 네비게이션 (position: fixed)

- **스크롤 진행률 반영**: 현재 섹션 카드가 진행률에 따라 확장
- **활성 인디케이터**: 확장된 카드 우측에 점(●) 표시
- **클릭 시**: 해당 섹션 첫 슬라이드로 smooth scroll
- **호버 시**: 약간 확대 + 섹션명 툴팁 (기존 유지)

### 삭제된 기능
- ~~좌우 화살표 버튼~~ → 스크롤로 대체
- ~~키보드 좌우 화살표~~ → 스크롤로 대체
- ~~"You are now entering..." 알림~~ → 제거

## 애니메이션 - v2

### 스크롤 스냅

- `scroll-snap-type: y mandatory` 적용
- 각 슬라이드에 `scroll-snap-align: start`
- CSS 기반 스냅 (Framer Motion 불필요)

### 섹션 진입 애니메이션

- **Intersection Observer** 사용
- 슬라이드가 뷰포트에 처음 진입할 때만 애니메이션 실행
- 텍스트, 이미지, 표 등이 순차적으로 나타남 (staggered)
- 한 번 표시된 후에는 즉시 표시

### 좌측 카드 확장

- 스크롤 위치에 따라 카드 높이 변경
- Framer Motion spring 애니메이션 (부드러운 전환)

| 요소 | 설명 |

### 섹션 전환 시스템

```
┌─────────────────────────────────────────────────────────────────┐
│  You are now entering ( Work ) section              ● 03 / 01  │
└─────────────────────────────────────────────────────────────────┘
```

- 섹션 진입 시 알림 메시지
- 페이지 인디케이터: "● 03 / 23" (섹션 내 현재 페이지 / 전체 페이지)
- 프로젝트명 라벨: "(7-Eleven Mobile)"

### 케이스 스터디 내부 구조

1. **헤더 영역**
   - 프로젝트명 (좌) + 카테고리 (중앙) + 클라이언트 로고 (우)
   - 검정 배경, 흰 텍스트
   - 핑크 핵심 설명 텍스트 (1-2줄 요약)

2. **히어로 일러스트**
   - 풀폭 아이소메트릭 일러스트레이션
   - 브랜드 특성을 시각적으로 표현

3. **타이틀 섹션**
   - 대형 타이포그래피 (브랜드 로고 + 캐치프레이즈)
   - 색상 블록 전환 (핑크 배너 + 검정 로고 + 핑크 배너)

4. **콘텐츠 그리드**
   - 모바일 앱 스크린샷 3개 (색상 카드 배경: 보라, 주황, 코랄)
   - 각 카드가 앱의 다른 기능 하이라이트

5. **텍스트 섹션** (Problem + Context)
   - 대형 타이틀: "A More Demanding Audience"
   - 2컬럼 본문: 좌측 문제 상황 / 우측 해결 필요성
   - 단색 배경 (빨강, 청록 등)
   - 화살표 일러스트로 시각적 연결

6. **페이지 인디케이터**
   - 각 섹션 우하단: "(7-Eleven Mobile)" + "● 04 / 23"

### 기술 스펙

#### 스크롤 스냅 구현
```css
/* 컨테이너 */
.portfolio-container {
  scroll-snap-type: y mandatory;
  overflow-y: scroll;
  height: 100vh;
}

/* 각 슬라이드 */
.slide {
  scroll-snap-align: start;
  height: 100vh;
}
```

#### 좌측 네비게이션
- `position: fixed` 로 화면 고정
- Intersection Observer로 현재 섹션 감지
- 현재 섹션 카드: 높이 확장 (진행률 반영)
- 클릭 시: `scrollIntoView({ behavior: 'smooth' })`

#### 진입 애니메이션
- Intersection Observer + `data-animated` 속성으로 1회만 실행
- staggered animation은 기존 패턴 유지

### UI/UX 스펙

#### 슬라이드 레이아웃
- 각 슬라이드: `min-height: 100vh`
- 섹션별 배경색: MDX frontmatter의 `color` 사용
- 콘텐츠 영역: 테두리 유지, 중앙 정렬

#### 좌측 카드 확장 동작
```
섹션 진입 전: 카드 축소 상태 (기본 높이)
섹션 진입 시: 카드 확장 시작
섹션 진행 중: 진행률에 따라 카드 높이 증가
섹션 이탈 시: 카드 축소
```

#### 프로그레스 바
- 하단 고정 (position: fixed)
- 전체 페이지 스크롤 진행률 표시
- PDF 다운로드 버튼 우측 배치

### 제약 및 트레이드오프

| 항목 | 선택 | 근거 |
|------|------|------|
| 스크롤 vs 캐러셀 | 스크롤 | 브라우저 기본 UX와 일치, 모바일 친화적 |
| 스냅 단위 | 슬라이드 | 각 페이지 경계 명확히 인지 |
| 키보드 네비 | 제거 | 스크롤로 충분, 복잡도 감소 |
| 섹션 전환 알림 | 제거 | 좌측 카드 확장으로 충분히 전달 |
