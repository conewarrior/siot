# UX 디자인 포트폴리오 페이지 PRD

## 개요

현재 포트폴리오 사이트에 **UX 디자인 포트폴리오** 전용 페이지를 추가한다.

- **메뉴 위치**: 프로젝트 | **포트폴리오** | 소개
- **URL**: `/portfolio`
- **SEO**: `noindex` 처리 (검색엔진 노출 방지, 링크로만 접근)

---

## 목표

1. **채용 담당자/클라이언트**가 역량을 빠르게 파악할 수 있는 구조
2. **PDF 변환**이 용이한 인쇄 친화적 레이아웃
3. 블로그 글과 PDF 프로젝트를 **동일한 위계의 케이스 스터디**로 구성

---

## 핵심 가치 (결정적 요인)

포트폴리오를 보는 사람이 "이 사람과 함께 일해야겠다"고 결정하는 요인:

1. **문제 해결 과정의 논리성** - 왜 그런 결정을 했는지, 사고의 흐름
2. **실제 비즈니스 임팩트** - 수치로 증명되는 결과 (전환율, 매출 등)
3. **디자인 감각과 완성도** - 시각적으로 세련되고 디테일이 살아있는지

## 슬라이드 구성

### 전체 순서

```
1. 커버 (+ 목차 통합)
2. 프로젝트 1 (4-10장)
3. 프로젝트 2 (4-10장)
4. 프로젝트 3 (4-10장)
5. About Me (포트폴리오 전용 소개)
6. 에필로그 (감사 인사 + 연락처)
```

**프로젝트 순서**: 가장 자신있는 것 먼저

### 케이스 스터디 필수 섹션 (각 프로젝트당)

| 섹션 | 설명 | 예상 슬라이드 |
|------|------|--------------|
| **Problem + Context** | 무슨 문제였고, 왜 중요했는지 | 1장 |
| **Process + Decision** | 어떤 선택지가 있었고 왜 그걸 골랐는지 | 2-4장 |
| **Outcome + Metrics** | 결과물과 수치화된 성과 (표로 시각화) | 1-2장 |
| **Reflection + Learning** | 배운 점, 다르게 했을 것 | 1장 |

### 콘텐츠 형태

- **시각 자료 중심** + 충분한 설명
- **지표는 표로 시각화**
- 블로그 글과 PDF 프로젝트 모두 동일한 케이스 스터디 구조로 재구성

## therawmaterials.com 디자인 분석 (2025-01-13)

### 좌측 내비게이션

| 요소 | 설명 |
|------|------|
| **색상 카드** | 각 섹션마다 다른 배경색 (주황, 보라, 검정, 파랑, 핑크, 노랑) |
| **섹션 번호** | 00, 01, 02, 03... 형식으로 순서 표시 |
| **현재 위치** | 활성 섹션에 작은 점(●) 인디케이터 |
| **로고 변형** | 스크롤에 따라 로고 형태 변화 (RMN → 아이콘) |

### 섹션 전환 시스템

```
┌─────────────────────────────────────────────────────────────────┐
│  You are now entering ( Work ) section              ● 03 / 01  │
└─────────────────────────────────────────────────────────────────┘
```

- 섹션 진입 시 알림 메시지
- 페이지 인디케이터: "● 03 / 23" (섹션 내 현재 페이지 / 전체 페이지)
- 프로젝트명 라벨: "(7-Eleven Mobile)"

### 케이스 스터디 내부 구조

1. **헤더 영역**
   - 프로젝트명 (좌) + 카테고리 (중앙) + 클라이언트 로고 (우)
   - 검정 배경, 흰 텍스트
   - 핑크 핵심 설명 텍스트 (1-2줄 요약)

2. **히어로 일러스트**
   - 풀폭 아이소메트릭 일러스트레이션
   - 브랜드 특성을 시각적으로 표현

3. **타이틀 섹션**
   - 대형 타이포그래피 (브랜드 로고 + 캐치프레이즈)
   - 색상 블록 전환 (핑크 배너 + 검정 로고 + 핑크 배너)

4. **콘텐츠 그리드**
   - 모바일 앱 스크린샷 3개 (색상 카드 배경: 보라, 주황, 코랄)
   - 각 카드가 앱의 다른 기능 하이라이트

5. **텍스트 섹션** (Problem + Context)
   - 대형 타이틀: "A More Demanding Audience"
   - 2컬럼 본문: 좌측 문제 상황 / 우측 해결 필요성
   - 단색 배경 (빨강, 청록 등)
   - 화살표 일러스트로 시각적 연결

6. **페이지 인디케이터**
   - 각 섹션 우하단: "(7-Eleven Mobile)" + "● 04 / 23"

## 디자인 개선 제안

### 적용할 요소

1. **색상 내비게이션 카드**
   - 현재 PRD의 좌측 메뉴를 색상 카드 스타일로 변경
   - 각 프로젝트/섹션마다 고유 색상 부여
   - 호버 시 확대 효과

2. **섹션 전환 표시**
   - "You are now entering (프로젝트명) section" 메시지 추가
   - 페이지 인디케이터 "● 01 / 05" 형식 도입

3. **케이스 스터디 헤더**
   - 프로젝트명 + 카테고리 + 역할 표시
   - 핵심 한 줄 요약 (핑크 강조)

4. **색상 블록 섹션**
   - 각 케이스 스터디 섹션마다 다른 배경색
   - Problem(빨강) → Process(청록) → Outcome(보라) 등

5. **대형 타이포그래피**
   - 섹션 타이틀을 화면 가득 채우는 큰 텍스트
   - 브랜드/프로젝트의 핵심 메시지 강조

### 보류할 요소

- **아이소메트릭 일러스트**: 제작 리소스 필요, 추후 고려
- **스크롤 기반 오브젝트 애니메이션**: 복잡도 증가, MVP 이후 고려
- **로고 모핑**: 현재 사이트 스타일과 맞지 않음
