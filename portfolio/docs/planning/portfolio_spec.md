## 프로젝트 2: 라벨링 툴 단축키 UX 개선

### 기본 정보
- **회사**: 셀렉트스타
- **기간**: 2022.09 - 2022.11
- **역할**: 프로덕트 디자이너 (전체 과정)

### 문제
- **비즈니스 임팩트**: 연 4.2억 작업비, 전체 매출의 20% 비중
- OKR: 프로젝트 원가 비용 절감 → 작업속도 향상
- 작업속도 10% 향상 시 4,200만원 절감 가능

### 해결 과정
**리서치 방법**:
- Mission Study (사내 스터디그룹)
- User Feedback (채널톡)
- Action Flow 분석

**가설**: 단축키 활용도를 높이면 가장 큰 개선

**솔루션 (AS-IS → TO-BE)**:
- 좌측 고정 툴바 → 플로팅 툴바
- 단축키 숨김 → 단축키 직접 노출
- 가이드 문서 필요 → 도움말 버튼 추가

### 결과 (Before/After 비교 측정)
- 주요 단축키 사용빈도 33% 증가
- 작업속도 11.57% 향상
- 클래스 선택 속도 4.53초 → 3.12초

### 회고
- **잘한 점**: (구체화 필요)
- **개선할 점**: (구체화 필요)

### 필요한 다이어그램 (PDF 전체 변환)
- 단축키 사용률 비교 차트 (1차 vs 2차)
- AS-IS / TO-BE 툴바 비교
- Action Flow 다이어그램
- 기타 PDF 내 모든 시각 자료

## 프로젝트 3: 패스트캠퍼스 디자인 시스템

### 기본 정보
- **회사**: 데이원컴퍼니 (패스트캠퍼스)
- **기간**: 2023.12 - 2024.03
- **역할**: UX UI 디자인 시니어 디자이너 (전체 과정)

### 문제
- 기존 디자인 가이드 빈약
- 재사용성 낮음, 규칙 부재
- 개발팀 모듈화 작업 니즈

### 해결 과정
**컬러 시스템**:
- WCAG 기반 명도대비율로 균일한 컬러 스케일 제작
- 다크모드 대응

**토큰 네이밍**:
- 기존: 위치 기반 (애매한 분류)
- 개선: Semantic 용도 기반 (Structural, Interactive, Overlay, Indicator)

**명세 규칙**:
- 어노테이션 킷 제작
- 명세 템플릿 (히스토리 기록)

**컴포넌트**:
- Auto Layout + Properties 활용
- 마스터 컴포넌트 라이브러리화

### 결과 (실측정 데이터)
- 컬러 스케일 10단계 체계화
- Variables + Storybook 도입
- 작업속도 약 20% 단축

### 회고
- **잘한 점**: (구체화 필요)
- **개선할 점**: (구체화 필요)
- **연계 콘텐츠**: 브런치 글 6개 (링크 연결)

### 필요한 다이어그램 (PDF 전체 변환)
- 컬러 스케일 시각화 (Gray, Blue, Green, Red, Yellow)
- 토큰 네이밍 구조도 (Group → Description → Item)
- 명세 템플릿 예시
- 기타 PDF 내 모든 시각 자료
