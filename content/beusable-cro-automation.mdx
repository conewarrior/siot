## 매일 반복되는 분석 작업

랜딩페이지 전환율 최적화(CRO)를 위해 뷰저블 히트맵 분석을 매일 확인하고 있었다.

문제는 다음과 같았다.

1. 뷰저블 로그인 → 리포트 확인 → 데이터 분석까지 매번 10분 이상 소요
2. 히트맵 데이터만 보면 "그래서 뭘 바꿔야 하지?" 답이 안 나옴
3. 분석 결과를 팀원들과 공유하려면 또 시간 소요

Claude Code가 브라우저도 조작할 수 있으니 이걸 자동화할 수 있지 않을까 생각했다. 명령어 한 줄로 뷰저블에서 히트맵 데이터 자동 수집, AI가 CRO 분석 보고서 생성, 이메일로 자동 발송까지 되는 시스템을 만들기로 했다.

## 시스템 구조

![CRO 자동화 파이프라인](/blog/cro-automation-pipeline.svg)

4개의 Python 모듈로 구성했다.

| 모듈 | 역할 | 기술 |
|------|------|------|
| beusable_scraper.py | 뷰저블 로그인 + 데이터 수집 | Playwright |
| landing_analyzer.py | 랜딩페이지 섹션 분석 | Playwright |
| report_generator.py | CRO 분석 보고서 생성 | Groq (Llama 3.3) |
| email_sender.py | 이메일 전송 | Gmail SMTP |

## 구현 과정

### Playwright로 뷰저블 로그인

처음엔 셀렉터가 안 맞아서 로그인이 계속 실패했다. Claude Code로 실제 브라우저를 열어 DOM 구조를 확인했다.

```python
await page.goto('https://www.beusable.net/signin')
inputs = await page.query_selector_all('input')
for inp in inputs:
    name = await inp.get_attribute('name')
    placeholder = await inp.get_attribute('placeholder')
    print(f'name={name}, placeholder={placeholder}')
```

결과:
```
name=email, placeholder=이메일
name=password, placeholder=비밀번호
```

이걸 보고 셀렉터를 수정했다.

```python
await self.page.locator('input[placeholder="이메일"]').first.fill(self.email)
await self.page.locator('input[placeholder="비밀번호"]').first.fill(self.password)
await self.page.locator('a.form__contents__button:has-text("로그인")').click()
```

### PV/UV 데이터 추출

처음엔 PV/UV가 0으로 나왔다. 다시 브라우저를 열어 확인하니 `.total-pv` 클래스에서 PV를 추출해야 했다. UV는 Analytics 페이지에 있어서 페이지 이동 후 추출하도록 수정했다.

### Gemini → Groq으로 변경

처음엔 Gemini API를 사용했는데 quota 초과 에러가 계속 발생했다.

```
429 RESOURCE_EXHAUSTED. limit: 0, model: gemini-2.0-flash
```

무료 API인 Groq으로 변경했다. Llama 3.3 70B 모델이 무료로 사용 가능하고 속도도 빠르다.

```python
from groq import Groq

client = Groq(api_key=api_key)
response = client.chat.completions.create(
    model="llama-3.3-70b-versatile",
    messages=[
        {"role": "system", "content": "당신은 전문 CRO 분석가입니다."},
        {"role": "user", "content": prompt}
    ]
)
```
